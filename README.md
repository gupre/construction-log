# 🏗️ Журнал работ

Внутренний инструмент для фиксации выполненных работ на строительном объекте. Прораб ведёт ежедневный учёт: что сделано, кем и в каком объёме.

**🔗 Демо:** https://construction-log-production.up.railway.app

---

## Стек и почему именно он

| Слой | Технология | Почему |
|------|-----------|--------|
| **Frontend** | React 18 + TypeScript | Требование ТЗ |
| **Сборка** | Vite | В разы быстрее CRA, нет лишнего boilerplate |
| **Запросы к API** | TanStack Query v5 | Кеширование, мутации и re-fetching из коробки — не нужен Redux |
| **Формы** | React Hook Form | Минимальный re-render, встроенная валидация без лишних зависимостей |
| **Backend** | Node.js + Express + TypeScript | Быстрый старт, единый язык с фронтендом |
| **ORM** | Prisma | Type-safe запросы, читаемые миграции, автогенерация клиента |
| **БД** | PostgreSQL 16 | Надёжная реляционная БД, хорошо работает с Prisma |
| **Контейнеры** | Docker + docker-compose | Воспроизводимое окружение, один команда для запуска |

## Запуск

### Требования
- [Docker Desktop](https://www.docker.com/products/docker-desktop)

### Два шага

```bash
git clone https://github.com/gupre/construction-log.git
cd construction-log
docker compose up --build
```

Открыть **http://localhost** — готово.

При первом запуске автоматически применяются миграции и загружаются тестовые данные: 18 видов работ и 5 примеров записей.

## Функциональность

**Обязательная:**
- Список записей: дата, вид работ, объём + единица измерения, ФИО исполнителя, примечания
- Фильтрация по диапазону дат (от / до)
- Сортировка по дате (новые / старые)
- Добавление записи с валидацией всех обязательных полей
- Редактирование записи
- Удаление с подтверждением
- Данные хранятся в PostgreSQL, фронтенд общается с бэкендом через REST API

**Дополнительная:**
- Справочник видов работ — отдельная таблица в БД, не хардкод
- Выбор вида работ из выпадающего списка вместо свободного ввода
- Единица измерения подставляется автоматически при выборе вида работ
- Страница управления справочником (добавить / удалить вид работ)
- Защита от удаления вида работ, используемого в существующих записях

## API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/entries` | Список записей (`dateFrom`, `dateTo`, `sortOrder`) |
| POST | `/api/entries` | Создать запись |
| PUT | `/api/entries/:id` | Обновить запись |
| DELETE | `/api/entries/:id` | Удалить запись |
| GET | `/api/work-types` | Список видов работ |
| POST | `/api/work-types` | Добавить вид работ |
| DELETE | `/api/work-types/:id` | Удалить вид работ |
| GET | `/api/health` | Health check |
